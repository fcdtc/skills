# 配色与字体(两种范式共用)

## 字体栈

```css
/* 中文 */
font-family: -apple-system, "PingFang SC", "Helvetica Neue", sans-serif;

/* 等宽(代码/数据/术语) */
font-family: "JetBrains Mono", "SF Mono", Menlo, monospace;
```

## 通用色板

| 系列 | 语义角色 | 色值 |
|------|---------|------|
| 橙系 | 主色 / 写入 / 角色A "动/写/暖" | `#c75a1f` `#d97706` `#ff8a3d` `#f59e0b` |
| 天青蓝 | 角色B "静/读/冷"(跟橙形成冷暖对比) | `#38bdf8` (sky-400) `#60a5fa` (blue-400) |
| 蓝系 | input / 客户端(区别于天青) | `#2c5d8c` `#2563eb` `#3b82f6` |
| 绿系 | 命中 / 正面 / 省钱 / ✓ | `#059669` `#10b981` `#2d8f5a` |
| 红系 | miss / × / 冷启动 / 警告 | `#dc2626` `#f87171` `#5a1f1a` |
| 黑 | 关键术语 pill 背景 | `#1a1a1a` |
| 灰 | **只做装饰底**(注释字、默认底色) | `#888` `#aaa` |

**铁律**:每种色在一个HTML里要有**固定语义**,不要随机换。如绿色一旦代表"命中"就不能在另一个图里代表别的东西。

## 冷暖对比的典型搭配

| 对立类型 | 暖侧 | 冷侧 |
|---------|------|------|
| 角色对立(静vs动、读vs写、客户端vs服务端、冷启动vs命中、查询vs事务) | 暖橙 `#ff8a3d` | 天青蓝 `#38bdf8` |
| 状态成败(命中vs错过、成功vs失败) | 绿 `#10b981` | 红 `#f87171` |

**灰色不参与"对立"**,只做不重要的背景元素(dot 默认底、注释字)。

## 选范式速查

| 主题特征 | 范式 | 主题色 | 参考文件 |
|---------|------|--------|---------|
| 数字、累积、前后对比、多状态演变(如缓存命中、token分布、调用次数、表扫描次数) | **dark 数据型** | 黑底 `#0f0f0f` | `style-dark.md` |
| 是什么、谁负责什么、流程怎么走、角色分工(如协议握手、消息流向、契约对仗) | **warm 概念型** | 奶油 `#fdf6ec` | `style-warm.md` |

**两种都不像?** 默认选 dark 数据型,因为它的 tab 结构能装下更多变化。如果实在是单一静态概念图,warm 更克制。
